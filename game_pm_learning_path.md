# 游戏产品经理学习路线图

**目标**: 成为合格的游戏产品经理  
**周期**: 3-6个月  
**前提**: 无需编程基础，但需要游戏经验

---

## 第一阶段：基础认知（2周）

### 1.1 了解游戏行业
**前提**: 无  
**实践**:
- [ ] 每天玩1款不同类型手游（SLG/卡牌/消除/竞技各1款）
- [ ] 记录每款游戏的：核心玩法、付费点、留存机制
- [ ] 关注TapTap/好游快爆热榜，了解国内主流品类

**产出**: 1份游戏品类分析报告（涵盖5种以上品类）

---

### 1.2 掌握游戏专业术语
**前提**: 无  
**实践**:
- [ ] 背诵以下术语并理解含义：
  - LTV、ARPU、ARPPU、付费率、留存率（次留/七留/月留）
  - Core Loop（核心循环）
  - DAU/MAU/PU
  - 付费渗透率
  - 流失率

**产出**: 术语表（自己的理解+实际案例）

---

## 第二阶段：工具技能（3周）

### 2.1 原型设计 - Figma
**前提**: 无  
**实践**:
1. 注册Figma账号（免费版即可）
2. 观看官方入门教程（30分钟）
3. 临摹1款游戏的UI界面（参考王者荣耀/原神设置页）
4. 制作1个最简单的功能原型（登录流程）

**产出**: 1个可交互的Figma原型文件

---

### 2.2 数据分析 - Excel
**前提**: 基础数学能力  
**实践**:
1. 学习VLOOKUP、数据透视表、图表制作
2. 下载任意游戏数据练习（可用模拟数据）
3. 计算：次留率、七留率、付费率、ARPPU

**产出**: 1份数据分析报表（含趋势图/漏斗图）

---

### 2.3 SQL基础（可选但推荐）
**前提**: 2.2完成  
**实践**:
1. 在W3Schools学SQL基础（3小时）
2. 用LeetCode数据库题目练习（5题）
3. 申请SQLite在线练习

**产出**: 10条SQL查询语句模板

---

### 2.4 流程图 - XMind/Miro
**前提**: 无  
**实践**:
1. 用XMind画出任意游戏的主界面结构
2. 用Miro画一个完整的用户注册流程

**产出**: 2张流程/结构图

---

## 第三阶段：产品思维（4周）

### 3.1 需求分析
**前提**: 第一阶段完成  
**实践**:
- [ ] 选择1款游戏，分析其"新手引导"模块
- [ ] 列出：用户目标 → 引导步骤 → 优化点
- [ ] 写出1份需求分析报告

**模板**:
```
需求背景: 
需求描述: 
目标用户: 
预期效果: 
评估指标: 
```

**产出**: 3份需求分析文档

---

### 3.2 竞品分析
**前提**: 无  
**实践**:
1. 选择2款同品类竞品
2. 从以下维度对比分析：
   - 核心玩法差异
   - 付费模式
   - UI/UE
   - 运营活动
3. 总结优劣势与机会点

**产出**: 1份竞品分析报告（建议用PPT格式）

---

### 3.3 数据分析实战
**前提**: 2.2完成  
**实践**:
1. 选择1款游戏
2. 收集公开数据（TapTap评价/七麦数据）
3. 分析：
   - 评分趋势
   - 用户留存评价
   - 付费评价

**产出**: 1份数据诊断报告

---

## 第四阶段：项目经验（4周）

### 4.1 独立完成1个小项目
**前提**: 前三阶段完成  
**实践**:
从以下选项选1个：

**选项A: 游戏策划案**
- 主题：设计1个小游戏（3天可完成）
- 文档包含：玩法描述、核心循环、UI结构、商业化思路

**选项B: 产品需求文档（PRD）**
- 主题：为现有游戏设计1个功能（如：签到系统/好友系统）
- 使用Notion或飞书文档

**选项C: Demo制作**
- 用Figma制作高保真原型
- 或用Canva制作可交互H5

**产出**: 1个完整的项目作品集

---

### 4.2 模拟面试
**前提**: 4.1完成  
**实践**:
1. 整理作品集（4.1产出）
2. 准备常见问题答案：
   - "请介绍你做的最成功的功能"
   - "如何提升次留？"
   - "如何设计付费点？"
   - "你玩过哪些游戏？"
3. 可以找朋友模拟或自己录音复盘

**产出**: 自我介绍 + 3个项目案例讲解

---

## 第五阶段：进阶提升（持续）

### 5.1 行业深度
- [ ] 每周阅读1篇游戏行业报告（Dataeye/App Annie）
- [ ] 关注GDC/游戏开发者大会分享
- [ ] 加入游戏行业社群

### 5.2 商业化理解
- [ ] 分析10款以上游戏的付费设计
- [ ] 理解：首充/礼包/月卡/ BattlePass设计逻辑

### 5.3 技术理解（加分项）
- [ ] 了解Unity/Unreal基础概念
- [ ] 理解程序实现难度，能评估需求可行性
