### Q2: 如何提升搜索精准度？

#### 可能的追问方式：
- "具体怎么做query理解？"
- "排序算法有哪些？"
- "如何衡量排序效果好不好？"
- "冷启动问题怎么解决？"
- "如果算法团队说做不了怎么办？"

#### 回答框架：

```xml
【整体思路】从4个维度提升搜索精准度：

【1. Query理解 - 理解用户意图】
- 意图识别：用户想搜什么？（问题？文档？人？）
- 实体提取：从query中提取关键实体
- 同义词扩展：用户说"电脑"可能想搜"计算机"
- 纠错能力：自动纠正拼写错误

【2. 召回阶段 - 扩大相关结果】
- 关键词召回：倒排索引
- 向量召回：语义相似度
- 知识图谱召回：关联实体

【3. 排序阶段 - 让好结果排前面】
- LTR（Learning to Rank）
- 商业价值因素（付费内容优先？最新内容优先？）
- 个性化因素（用户历史行为）

【4. 反馈闭环 - 持续优化】
- 显式反馈：用户点不点搜结果
- 隐式反馈：停留时长、滚动深度
- 人工标注：bad case修正

【如果算法说做不了】
- 分阶段：先做简单方案，v1再迭代
- 找替代：规则+算法结合
- 降级策略：保证基本可用
```

#### 需要提前准备的知识点：

| 知识点 | 了解程度 | 建议 |
|--------|----------|------|
| Query解析 | 理解 | query理解流程 |
| 倒排索引 | 理解 | 搜索引擎基础 |
| 向量检索 | 了解 | 语义搜索原理 |
| LTR | 了解 | 排序学习基础 |
| 评估指标 | 掌握 | MAP、NDCG |

### Q3: 如何衡量AI产品的成功？

#### 可能的追问方式：
- "这些指标怎么量化？"
- "指标之间冲突怎么办？"
- "短期指标和长期指标哪个重要？"
- "如果指标都很好但用户不买单怎么办？"

#### 回答框架：

```xml
【指标体系】分4个层次：

【1. 技术指标】
- 搜索准确率：Top3/Top5准确率
- 搜索召回率：相关结果召回比例
- 问答准确率：正确答案率
- 响应延迟：P99 < 200ms

【2. 用户指标】
- 搜索无结果率：< 5%
- 点击率（CTR）：搜索结果点击比例
- 问答解决率：用户问题被正确回答的比例
- 停留时长：深度使用情况

【3. 业务指标】
- 知识覆盖率：知识库覆盖范围
- 知识更新时效：新内容入库时间
- 用户满意度：NPS

【4. 商业指标】（如果有）
- 付费转化率
- 广告点击收入
- 会员续费率

【指标优先级】
先看技术指标（基础可用）
再看用户指标（体验好坏）
最后看业务指标（商业价值）

【指标冲突时】
- 排序：业务指标 > 用户指标 > 技术指标
- 例：准确率高但用户不用 = 没用
```

#### 需要提前准备的知识点：

| 知识点 | 了解程度 | 建议 |
|--------|----------|------|
| 搜索评估指标 | 掌握 | Precision@K, Recall@K, MAP, NDCG |
| 问答评估指标 | 了解 | EM, F1 |
| 数据分析 | 熟练 | SQL, Excel, Python |

### Q: 你如何理解"AI产品经理"和"传统产品经理"的区别？

#### 可能的追问方式：
- "AI产品有什么特殊性？"
- "AI产品经理需要懂算法吗？"
- "怎么和算法工程师合作？"
- "AI产品有哪些坑？"

#### 回答框架：

```xml
【核心区别】

| 维度 | 传统产品 | AI产品 |
|------|----------|--------|
| 不确定性 | 需求明确 | 效果不确定 |
| 迭代方式 | 敏捷迭代 | 数据驱动+算法迭代 |
| 评估指标 | 确定性指标 | 概率性指标 |
| 技术依赖 | 较低 | 非常高 |

【AI产品的特殊性】
1. 效果是概率性的：不能100%保证准确
2. 依赖数据质量：garbage in, garbage out
3. 冷启动难：没有数据就效果差
4. 解释性差：为什么这么推荐不一定说得清

【AI产品经理的能力要求】
1. 理解技术原理：不用会写代码，但要懂能做什么
2. 数据敏感：会用数据发现问题
3. 降预期能力：管理业务方对AI的预期
4. 场景选择：找到AI能发挥价值的场景

【怎么和算法工程师合作】
1. 需求对齐：说清楚要解决什么问题
2. 评估可行：让技术评估工程量
3. 分阶段交付：先上线再看效果
4. 持续优化：基于数据迭代
```
