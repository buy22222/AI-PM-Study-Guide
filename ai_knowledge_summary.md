## 1. RAG架构（检索增强生成）

### 是什么
RAG = Retrieval Augmented Generation（检索增强生成）
简单说：先搜索相关内容，再让AI基于这些内容生成答案

### 学习资源

**推荐网站**：
1. **LangChain官方文档** 
   https://python.langchain.com/docs/get_started/introduction
   
2. **RAG入门教程（英文）**
   https://github.com/ageron/handson-ml3/blob/main/14_loading_and_fine_tuning_transformers.ipynb
   
3. **中文RAG介绍**
   https://www.promptengineer.com/blog/rag-tutorial/

### 核心概念（了解即可）

| 概念 | 解释 |
|------|------|
| 索引 | 把文档切成小块，存入向量数据库 |
| 检索 | 用户提问时，找相似的小块 |
| 生成 | 把检索结果给LLM生成答案 |

### 面试话术

> RAG是一种架构，核心是"搜+生成"。先把知识库索引，用户提问时先搜索相关内容，然后把搜到的内容给大模型，让它基于这些内容回答。这样可以解决大模型知识过时、幻觉的问题。

## 2. 向量数据库

### 是什么
专门存储"向量"的数据库，用于相似度搜索

### 学习资源

**推荐网站**：
1. **Milvus官方文档（推荐）**
   https://milvus.io/docs/overview.html
   
2. **Pinecone官方**
   https://www.pinecone.io/learn/

3. **向量检索入门（中文）**
   https://zhuanlan.zhihu.com/p/594870375

### 核心概念

| 概念 | 解释 |
|------|------|
| 向量 | 把文字/图片转成一串数字 |
| 相似度 | 数字越接近 = 意思越接近 |
| 索引 | 加速搜索的数据结构 |

### 面试话术

> 向量数据库是用来做语义搜索的。比如用户搜"手机"，传统搜索只能匹配"手机"，但向量数据库可以找到"iPhone"、"智能手机"这些相关的结果，因为它理解意思而不是匹配关键词。

## 3. 推荐系统

### 是什么
根据用户行为，推荐可能感兴趣的内容

### 学习资源

**推荐网站**：
1. **推荐系统入门（英文）**
   https://www.youtube.com/watch?v=9gBC9RheA8I
   
2. **协同过滤解释**
   https://www.ibm.com/cloud/learn/collaborative-filtering
   
3. **中文推荐系统科普**
   https://zhuanlan.zhihu.com/p/26626755

### 核心概念

| 概念 | 解释 |
|------|------|
| 协同过滤 | "和你相似的人也喜欢" |
| 内容推荐 | "你喜欢这个，也喜欢那个" |
| 冷启动 | 新用户没数据怎么推荐 |

### 面试话术

> 推荐系统核心是预测用户会不会点击/购买。常见方法一是协同过滤，A和B都看过电影A1，如果B看了电影B1，就给A推荐；方法二是内容推荐，如果你之前看过科幻电影，就推荐更多科幻。

## 4. NLP基础（自然语言处理）

### 是什么
让计算机理解人类语言

### 学习资源

**推荐网站**：
1. **NLP入门（英文）**
   https://www.kaggle.com/learn/natural-language-processing
   
2. **Hugging Face Transformers**
   https://huggingface.co/learn

3. **中文NLP资源**
   https://github.com/DSXiangLi/Interview

### 核心概念

| 概念 | 解释 |
|------|------|
| 分词 | "我爱你" → "我/爱/你" |
| 实体识别 | 识别人名、地名等 |
| 意图识别 | 理解用户想做什么 |
| 情感分析 | 判断正面/负面情绪 |

### 面试话术

> NLP是让机器理解人话。比如"我想买手机"这句话，分词会切成"我/想/买/手机"，实体识别会认出"手机"是产品，意图识别会判断用户想"购买"。这些是搜索和问答系统的基础。

## 5. LLM（大语言模型）

### 是什么
能理解和生成文字的AI模型

### 学习资源

**推荐网站**：
1. **LLM入门（英文）**
   https://www.deeplearning.ai/short-courses/
   
2. **Prompt Engineering指南**
   https://www.prompteng.io/
   
3. **Sam Altman关于GPT的演讲**
   https://www.youtube.com/watch?v=AK7cLhvh6b4

### 核心概念

| 概念 | 解释 |
|------|------|
| Token | 模型处理文字的最小单位 |
| Temperature | 控制回答的随机性 |
| System Prompt | 给AI的角色设定 |

### 面试话术

> 大语言模型本质是"下一个词预测"，给定前面的词，预测下一个最可能的词。它不需要死记硬背，而是理解语言模式，所以可以回答没见过的問題。Prompt是给模型的指令，好的prompt能让模型输出更准确。

## 6. Fine-tuning（微调）

### 是什么
在已有模型基础上，用特定数据训练

### 学习资源

**推荐网站**：
1. **Fine-tuning入门**
   https://www.deeplearning.ai/short-courses/
   
2. **LLaMA微调教程**
   https://github.com/tloen/alpaca-lora

### 核心概念

| 概念 | 解释 |
|------|------|
| 预训练 | 用海量数据训练基础模型 |
| 微调 | 用少量数据让模型适应特定任务 |
| 训练成本 | 非常高（需要GPU） |

### 面试话术

> 微调是在已经训练好的模型基础上，用特定数据让它更适应某个场景。比如通用GPT可能不太懂我们公司产品，但用公司FAQ微调后，它就能更准确地回答我们产品相关的问题。

## 8. 搜索评估指标

### 核心指标

| 指标 | 解释 | 公式 |
|------|------|------|
| Precision@K | 前K个结果里有多少相关 | 相关数/K |
| Recall@K | 所有相关结果找到了多少 | 找到的相关数/总相关数 |
| MAP | 排序质量的综合指标 | 平均准确率 |
| NDCG | 考虑排序位置的打分 |  |

### 面试话术

> 搜索效果主要看Precision和Recall。Precision@3=0.7是说搜到的3个结果里有70%是用户想要的。NDCG是更高级的指标，不仅看找没找到，还看排在第几名，排在前面比排在后面分数高。

## 面试中如何运用这些知识

### 被问到"你懂AI技术吗"

> 我是CS硕士背景，虽然不会写代码，但我理解AI产品背后的原理。比如我知道搜索产品核心是Query理解→召回→排序→重排，也了解RAG、向量检索这些常用架构。在之前的工作中，我能和算法团队进行有效沟通，把业务需求转化为技术方案。

### 被问到"怎么评估搜索效果好"

> 从4个维度：技术指标看准确率和召回率；用户指标看点击率和问答解决率；业务指标看转化率；最后看用户满意度NPS。实际工作中我们主要用NDCG来评估排序效果。

### 被问到"为什么选择AI产品"

> AI产品最有挑战性也最有前景。传统产品确定性高，但AI产品是概率性的，需要用数据驱动。需要产品经理既懂业务又懂技术，还能管理预期，这是我的优势。
